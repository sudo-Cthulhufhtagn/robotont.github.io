Demos on Gazebo¶

Launching the Simulation¶

  1. To launch the simulator:

    roslaunch robotont_gazebo gazebo_teleop_keyboard.launch

There are two arguments on the launch file, that change the map and the position, where the robot will spawn:

  • world - the map where the robot will spawn, the default map is empty world
  • x_pos - x coordinate of the map, controls where the robot will spawn

For example, the following command will spawn the robot to a map called minimaze.world in position x=2:

roslaunch robotont_gazebo gazebo_teleop_keyboard.launch world:=$(rospack find robotont_gazebo)/worlds/bangbang.world x_pos:=2

Maps¶

  1. minimaze.world

    To run

    roslaunch robotont_gazebo gazebo_world_minimaze.launch

  2. bangbang.world

    To run

    roslaunch robotont_gazebo gazebo_world_between.launch

  3. between.world

    To run

    roslaunch robotont_gazebo gazebo_world_bangbang.launch

2D Mapping¶

gmapping¶

  1. Launch the simulator

    roslaunch robotont_gazebo gazebo_world_minimaze.launch

  2. Launch the gmapping software and visualize the map on RViz

    roslaunch robotont_gazebo gazebo_gmapping.launch

  3. Drive around using the keyboard and map the world

Google Cartographer¶

  1. Launch the simulator

    roslaunch robotont_gazebo gazebo_world_minimaze.launch

  2. Launch Google Cartographer software and visualize the map on RViz

    roslaunch robotont_gazebo 2d_nav_carto_gazebo.launch

  3. Drive around using the keyboard and map the world

ROS navstack¶

  1. Using the navstack in ROS is very straightforward, you tell the robot where it is (if it doesnt already know) and where it needs to go.

  2. For setting initial pose, click on 2D Pose Estimate and drag the arrow where and how the robot actually is.

  3. To tell the robot where to go, click on 2D Nav Goal and drag the arrow to where you want the robot to go and which way does it have to face.

  1. From planner.yaml you can tune the parameters for the planner. Reference can be found here.
