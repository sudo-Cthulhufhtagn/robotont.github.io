Demos on Robotont¶

Before starting establish an ssh connection between the robot and the user PC as shown here: Connecting with the robot remotely.

The following packages are needed to run the demos:

  1. For 2D mapping:

    sudo apt update
sudo apt install ros-melodic-depthimage-to-laserscan
sudo apt install ros-melodic-cartographer-ros
sudo apt install ros-melodic-move-base

  2. For 3D mapping:

    sudo apt install ros-melodic-rtabmap-ros

  3. For AR tracking:

    sudo apt install ros-melodic-ar-track-alvar

2D mapping¶

  1. On Robotont on-board computer launch 2d_nav_carto.launch

    roslaunch robotont_demos 2d_nav_carto.launch

  2. On PC launch display_2d_mapping.launch to visualize the result

    roslaunch robotont_demos display_2d_mapping.launch

  3. To move the robot open another terminal window on the PC and run teleop twist keyboard

    roslaunch robotont_demos teleop_keyboard.launch

    Hint

    Notice that the teleop node only receives keypresses when the terminal window is active.

3D mapping¶

  1. On Robotont on-board computer launch 3d_mapping.launch

    roslaunch robotont_demos 3d_mapping.launch

  2. On PC launch display_3d_mapping.launch to visualize the result

    roslaunch robotont_demos display_3d_mapping.launch

  3. To move the robot open another terminal window on the user PC and run teleop twist keyboard

    rosrun teleop_twist_keyboard teleop_twist_keyboard.py __ns:=/robotont

    Hint

    Notice that the teleop node only receives keypresses when the terminal window is active.

AR tracking¶

  1. On Robotont on-board computer launch ar_follow_the_leader.launch (change tag_nr with your AR tag number)

    roslaunch roslaunch robotont_demos ar_follow_the_leader.launch marker_id:=tag_nr

  2. On PC launch display_ar_marker.launch to visualize the result

    roslaunch robotont_demos display_ar_marker.launch

  3. To move the robot open another terminal window on PC and run teleop twist keyboard

    roslaunch robotont_demos teleop_keyboard.launch

    Hint

    Notice that the teleop node only receives keypresses when the terminal window is active.

ROS navstack¶

  1. Using the navstack in ROS is very straightforward, you tell the robot where it is (if it doesnt already know) and where it needs to go.

  2. For setting initial pose, click on 2D Pose Estimate and drag the arrow where and how the robot actually is.

  3. To tell the robot where to go, click on 2D Nav Goal and drag the arrow to where you want the robot to go and which way does it have to face.

  1. From planner.yaml you can tune the parameters for the planner. Reference can be found here.
